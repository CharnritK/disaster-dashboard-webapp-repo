02 · Foundations

Color: a semantic namespace system

The current palette uses blue / amber / red somewhat interchangeably for AI, data quality, and status. This system splits color into named semantic roles. Each role owns a hue and a four-part token set — solid, text, surface, border — so AI, quality, and status never borrow each other's color. Color is never the only signal: every status pairs with a text label, and a badge shape or icon.

Contrast & usage rules

  • Text on white meets WCAG AA. Use --*-text tokens for any small text; the --*-solid hue is for fills, dots, and borders only.
  • Brand teal is sparing. Product identity, primary completion, and "ready" states only — not a generic accent.
  • Red is reserved. Only failed checks, unsafe charts, and blocked actions. Caution and ambiguity are amber, not red.
  • Never color-only. Pair every status hue with a label and a badge shape or icon (see §06–07).
  • Dark-ready. All semantic tokens are aliases — a future dark theme reassigns the four-part sets, not the component code.

03 · Foundations

Typography

Rethink Sans carries the whole interface — a humanist grotesque that reads as institutional and plain-spoken rather than corporate. It is paired with Spline Sans Mono for the things that must read as data: column names, field codes, metric values, and token references. The scale is restrained — six text sizes do all the work, so hierarchy comes from size and weight, not from many fonts.

07 · System

AI & governance state model

AI is an optional, server-side assist on top of a deterministic core. Every failure path — disabled, rate-limited, timed out, quota-exhausted — degrades to deterministic output and keeps the workflow moving. The interface states which mode produced each result, and never lets AI read as authoritative.

✓

Governance posture. Deterministic is the default and the floor. Fallback is normal, expected behavior — it is shown calmly in amber (review), not as a red error. Red is reserved for blocked actions and unsafe output.

08 · System

Data-visualization standards

Deterministic guardrails own chart type, sorting, safety, badges, and summaries — AI may suggest, but vizPolicy.ts decides. The contract: every chart carries a title, a rationale, a source/caveat note, a quality badge, and an accessible summary. No exceptions.

Required parts

  1. Title — what is measured, with unit/denominator.
  2. Subtitle — scope, count shown vs. total, normalization.
  3. Quality badge — Pass / Caution / Blocking, inherited from §06.
  4. Rationale — why this chart answers the decision.
  5. Source + caveat — files, transforms, known limits.
  6. SR summary — trend + range + flag, in words.

Map safety rules

vizPolicy.ts
✕Never do
  • Encode raw counts as choropleth area fill.
  • Draw boundary choropleths without verified local boundary geometry.
  • Call geocoders, basemaps, or remote boundary services from the browser.
  • Default to a map when geometry or quality is uncertain.
✓Safe path
  • Choropleth only with a normalized measure (rate, ratio, density, index) + verified geometry.
  • Coordinate maps are local uploaded-coordinate views only.
  • When a map is unsafe, fall back to a bar or table with a caveat — not silently.
  • Keep denominator fields visible whenever comparisons depend on scale.

Fallback in practice. “A choropleth of raw affected counts would mislead by area size. Showing a ranked bar chart of the rate instead.” — the caveat travels into the source note and the export package.

Accessible summaries & mobile

  • Every chart exposes its title via aria-labelledby and a text summary via aria-describedby.
  • Missingness shown as labelled values, never color alone.
  • On mobile, multi-series collapse to ranked bars or a table; pies become legends-with-values.
  • Bars keep value labels in-bar so the chart reads without axes.

Export / PDF-safe rendering

  • Exports preserve title, source, caveat, quality badge, and review notice on every chart.
  • Animations and tooltips are disabled in the export render target.
  • Fixed 1120px export width; shadows removed; colors flattened for print.
  • Status is legible in grayscale — shape + label carry meaning.
